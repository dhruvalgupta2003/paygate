# RFCs

Design proposals go here. Small changes land via PR + discussion; anything
that touches verification, signing, config, or public API shape lands as
an RFC first.

Format:

```
# RFC-<n>: <title>

- Status: draft | accepted | rejected | superseded
- Authors:
- Created: YYYY-MM-DD
- Supersedes / Superseded by:

## Motivation
## Proposal
## Alternatives
## Security considerations
## Compliance considerations
## Open questions
```
